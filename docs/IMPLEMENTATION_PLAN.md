# 바로쇼핑 (ShortsGen) — 구현 스텝 (Implementation Plan)

> `docs/PRD.md` v0.2의 로드맵(§13)·기능요구(§8 FR)·데이터모델(§9·§10)을 **실행 가능한 빌드 스텝(WBS)** 으로 분해한 문서.
> 스텝 ID = `P{phase}-{n}`. 상태 = ☐ 미착수 · ◐ 진행 · ☑ 완료.
> 현재 시작점: **Remotion 렌더 레이어만 구현(~15%)**, 나머지는 미구현. 모노레포·백엔드·DB·워커 전부 신규 구축 대상.

---

## 0. 한눈에 보기 (Critical Path)

```
[지금] Remotion 렌더 단독 동작
   │
P0  렌더 레이어 안정화 ──────────────┐ (현 코드 보강, 1~3일)
   │                                 │
P1  MVP 수직 슬라이스 ★최우선         │ 수동 데이터 → 렌더 → YouTube 수동 발행
   │  └ 여기서 "발행까지" 가치가 닫힘 (스크래퍼·큐 없이)
   │
P2  생성 자동화 ─── 모노레포→DB→백엔드/큐→스크립트→음성→렌더통합→스크래퍼→CLI
   │
P3  운영 자동화 ─── 대시보드연동→발행어댑터→VPS배포→n8n스케줄→Telegram
   │
P4  성과/최적화 ─── 성과수집→KPI→A/B→비용확정→캐시강화
```

**가장 먼저 할 일 3가지:** `P0-1`(렌더 동작 확인) → `P1-1`(상품 JSON 템플릿) → `P1-4`(1건 end-to-end 발행).

**선행 차단 항목(운영자 결정 대기):** `D3`(예산 금액)은 P4-4 전, `D1`(스크래핑 대상·권한)은 P2-7 착수 전 확정 필요 (PRD §15).

## Phase 0 — 렌더 레이어 안정화 (현 코드 기반)
> 목표: 기구현 Remotion 합성을 운영 가능 상태로 다지기. 신규 인프라 없음.

| ID | 작업 | 산출물 | 의존성 | 완료기준(DoD) | 근거 |
|---|---|---|---|---|---|
| P0-1 | 의존성 설치 · 로컬 렌더 동작 확인 | `npm run studio` 프리뷰 + `npm run render:props` → `out/video.mp4` | 없음 | 기본/예제 props로 무오류 렌더, 썸네일(`npm run still`) 생성 | §16 |
| P0-2 | 긴 상품명 텍스트 오버플로 자동 축소 | fit-to-width 유틸 + 적용 | P0-1 | 2줄/긴 이름이 박스 밖으로 안 넘침 | FR-4, `ProductScene.tsx` |
| P0-3 | Outro 타일 동적화 (현재 4개 고정) | `slice(0,4)` 제거·상품수 대응 레이아웃 | P0-1 | 상품 1~6개에서 타일 레이아웃 정상 | FR-4, `OutroScene.tsx:48` |
| P0-4 | (선택) 오디오 트랙 골격 | `schema.ts`에 audio 필드 + `<Audio>` 배선 | P0-1 | 오디오 없으면 무음 회귀 0, 있으면 재생 | FR-4, README:145 |
| P0-5 | 렌더 스모크 검증 루틴 | 1·3·6개 상품 샘플 JSON 렌더 스크립트 | P0-2~3 | 3개 샘플 전부 무오류·길이식 일치 | §9 길이식 |

## Phase 1 — MVP 수직 슬라이스 ★최우선
> 목표: **수동 상품 JSON → 렌더 → YouTube Shorts 수동 발행**까지 사람이 1회 관통. 스크래퍼·큐·백엔드 없이 가치 닫힘.

| ID | 작업 | 산출물 | 의존성 | 완료기준(DoD) | 근거 |
|---|---|---|---|---|---|
| P1-1 | 상품 JSON 입력 템플릿 + 작성 가이드 | `input-props` 기반 빈 양식 + 필드 설명 | P0 | 운영자가 가이드만 보고 1건 작성 가능 | §9.1, D1 |
| P1-2 | 렌더 실행 래퍼 | 입력 JSON → `out/{id}.mp4` + 썸네일 추출 스크립트 | P1-1 | 한 명령으로 영상+썸네일 산출 | FR-4 |
| P1-3 | YouTube Shorts 발행 체크리스트 | 제목/설명/태그/썸네일 규격·업로드 절차 문서 | 없음 | 수동 업로드 단계가 문서화됨 | D2, FR-8(수동) |
| P1-4 | **MVP 1건 end-to-end 검증** | 상품 1건 → 영상 → YouTube 발행 1건 | P1-1~3 | 실제 1건 발행 완료(전 과정 1회 통과) | §13 Phase1 DoD |

> ✅ P1-4 통과 시점 = "발행까지 닿는" 최소 제품 완성. 이후 Phase는 이 수동 흐름의 자동화.

## 횡단 관심사 (모든 Phase 공통)
- **시크릿**: `.env`(+`.env.example`)에만, 커밋·문서·채팅 금지. VPS에만 실 키. (§11 보안)
- **커밋 규약**: Conventional Commits, Biome(ESLint 아님), `any` 금지. (CLAUDE.md)
- **외부·비가역 행위**: 발행/메시지/삭제는 승인 게이트·사전 승인. (§12)
- **테스트**: 현재 테스트 0개 → 최소 워커 단위테스트·렌더 스모크부터 도입.
